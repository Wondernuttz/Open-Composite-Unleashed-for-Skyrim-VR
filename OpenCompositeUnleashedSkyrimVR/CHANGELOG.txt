# Changelog

## v2.0.0

Major overhaul of the VR keyboard input system — thumbstick navigation replaced with laser pointer aiming, and the keyboard is now world-anchored and repositionable.

### New Features

- **Laser pointer beams** — Each controller projects a visible warm-white beam onto the keyboard surface with a cursor dot showing exactly where you're aiming. Point and pull trigger to type.
- **World-anchored keyboard** — The keyboard now spawns in world space in front of the player (80cm forward, 35cm below eye level) and stays fixed in place. No longer head-locked by default.
- **Grab bar** — A dedicated strip at the top of the keyboard lets you hold trigger and drag to reposition the keyboard anywhere in your play space. The keyboard follows your laser 1:1 during drag.
- **MOVE / LOCK buttons** — MOVE (top-left) indicates the keyboard is in world-anchored mode and can be grabbed. LOCK (top-right) toggles head-locked mode where the keyboard follows your head. Both buttons show hover highlights when aimed at.
- **Clickable text cursor** — Tap anywhere in the text input bar to place the cursor at that character position. A blinking cursor (500ms on/off) shows your insertion point.
- **Cursor-based editing** — Characters now insert at the cursor position instead of always appending to the end. Backspace deletes the character before the cursor.
- **Beam billboard orientation** — Laser beams always face the viewer for maximum visibility regardless of viewing angle.

### Changes

- Thumbstick/DPad key navigation removed (laser pointers replace it entirely)
- Keyboard spawns facing the player based on head orientation at open time
- Selection highlight is now a unified cream-gold color for both controllers (replaces the old blue-left/green-right scheme)
- Text bar is now a distinct clickable zone separated from the key area
- Warm white color scheme for laser beams and cursor dots (RGB 255/240/220)

### Technical

- Laser beams rendered as XrCompositionLayerQuad overlays (one per hand)
- Ray-plane intersection for keyboard hit testing supports arbitrary keyboard orientation
- Grab uses ray-plane intersection against the original grab plane to prevent feedback lag
- Head-lock toggle switches between viewSpace and floorSpace for both keyboard and laser layers
- Swapchain and D3D11 resources for laser beams properly created and destroyed (no leaks)

## v1.0.0

Initial release of the VR keyboard for Skyrim VR via OpenComposite.

### Features

- Native OpenXR composition layer keyboard
- Head-locked positioning (keyboard follows head movement)
- Thumbstick navigation between keys
- Trigger to press keys, grip to dismiss
- Dual controller support (blue highlight left, green highlight right)
- Skyrim-themed dark stone and gold UI
- Shift, Caps Lock, Backspace, Done keys
- English (GB) layout
- Works with character creation, enchanting, and any SteamVR keyboard API call
